The Amnesia Test

Every AI coding tool is brilliant and forgets everything between sessions.
We ran the same task twice on the same codebase -- once on our last release, once on the new build -- to show the difference a fleet that remembers makes.
SAY: "This is a real head-to-head. Same repo, same two features, same machine. Only the build differs."

The tasks (identical for both)

Two sprints, back to back

  • Sprint 1 -- Note archiving: add an archived flag, archive/unarchive endpoints, exclude archived from the default listing.
  • Sprint 2 -- Pagination: paginate the listing -- and it must compose with the archived filter sprint 1 just built.
Sprint 2 deliberately touches the same listing code sprint 1 changed. That is where a fleet that remembers pulls ahead.
SAY: "The trap is sprint 2. It has to understand what sprint 1 did. A tool with no memory re-learns it from scratch."

The honest part

What this doesn't claim

We are not showing a "40% fewer tokens" headline. The two builds structure their work differently, and the sandbox cost telemetry didn't populate cleanly for either side.

The defensible, verified claim is the capability: reuse happened in Env B and is impossible in Env A. v0.3.4 can't even measure its own tokens -- its logs read Act tokens: 0.
SAY: "Credibility matters more than a big number. The win is real and it's about capability, not a cherry-picked benchmark."
The twist

The demo caught a release-blocker

The first Env B runs showed no knowledge activity. Root cause: the shipped agent definitions hard-coded tool lists that excluded every KB / code-intelligence tool -- so the standard flow's agents literally couldn't use the feature. The whole layer was inert in the real product.

Found it, fixed it, re-ran clean -- all 16 captures above are from the fixed run.
A rigorous head-to-head that catches your own bug before ship beats any favorable benchmark.
SAY: "This is the best part of the story. We didn't just measure a feature -- we caught it silently broken in the release path and fixed it in the same session."
Takeaway

A fleet that gets cheaper, more consistent, and more aligned the more it works

Knowledge captured once is retrieved forever. A correction made once propagates forever. A standing instruction given once applies forever -- and all of it lives in your repo, not a vector DB you rent.

v0.3.4 finished each sprint and forgot everything. The new build learned from it.
SAY: "That's the pitch. Same work, but the fleet compounds -- and it caught its own bug proving it."
