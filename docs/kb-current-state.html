3 -- WHAT NEEDS TO BE DONE (PRIORITY ORDER)
1
Enforce the CONFIRMED gate in code
kb_capture caps confidence at INFERRED regardless of what the caller passes. Only kb_promote -- called by the KB Agent holding the reviewer verdict -- can mint CONFIRMED. This makes "CONFIRMED = passed review" true by construction, not by prompt obedience. Touch: kb-capture.ts, sqlite-provider.ts.
2
Retire kb_harvest; make KB Agent direct capture the official path
The regex-over-transcript design cannot work (agents have no access to their own transcript). The KB Agent already reads diff + feedback.md + progress notes and captures directly -- codify that as the flow, remove harvest from templates.
3
Add kb_list + export to git (the Knowledge Bible)
New kb_list tool: dump all entries filterable by confidence / module / sprint. KB Agent commits CONFIRMED entries to .fleet/kb-canonical.json on the branch -- shared, versioned, visible. This answers "where do I see my learnings."
4
Wire staleness detection
0 entries marked stale after 4 sprint days of code change -- the file-hash sync never runs. Run it inside kb_session_prime so entries touching changed files get flagged before an agent trusts them.
5
Record KB Agent dispatch in a ledger
KB Agent runs on prompt convention -- if the PM skips it (observed before this week's patch), knowledge is silently lost. Log each dispatch in progress.json / beads so a missing run is visible.
6
Boost retrieval hit rate
5 of 41 entries ever used. After fixes 1-5 land, measure whether hint_symbols matching in kb_session_prime is too narrow; widen to module-level matching if entries still go unread.
