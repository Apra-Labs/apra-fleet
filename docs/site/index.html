Architecture

Three Roles. One Organization.

Fleet doesn't give you one smarter agent. It gives you the structure that makes engineering teams effective.

🧠
Strategist

PM Agent

Plans work, tracks progress, never writes code. Uses Opus for architecture decisions, Haiku for status checks. Knows when to think big and when to move fast.

🔨
Builder

Doer Agents

Execute on your machines — Windows, macOS, Linux. Commit, push, hit checkpoints. Follow the plan exactly. No improvisation, no drift.

🛡
Gatekeeper

Reviewer Agents

Independent verification with fresh context every review. No approval means no merge. Quality is structural, not aspirational.

PM Plans → Doer Builds → Reviewer Verifies → PM Deploys

The doer-reviewer pattern is the architecture — not a feature.

Built for Real Engineering

Quick Start

Up and Running in Minutes

Three steps from zero to a managed engineering fleet.

1

Install

Install from npm or grab a release from GitHub.

npm install -g apra-fleet
2

Register Members

Add machines to your fleet — local or remote via SSH.

register_member name=dev1 host=192.168.1.10
3

Start Managing

Plan work, assign agents, and ship code.

/pm plan → /pm start → /pm deploy
