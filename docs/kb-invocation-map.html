HUMAN (periodic)

The compounding loop: next sprint's planner calls kb_session_prime and receives everything above -- live CONFIRMED entries first, graph-expanded neighbors below, bible entries seeding any cold start. Knowledge captured once is retrieved forever; a correction made once propagates forever; a user instruction given once applies forever. Measured this session: 76% of entries retrieved, 235 total reads, doers skipping exploration on KB hits.

TRUST TIERS (enforced in code)

UNVERIFIED  harvest autowire output; transcript-only claims.
INFERRED  the capture ceiling for every agent -- kb_capture clamps here.
CONFIRMED  mintable only by kb_promote, gated on a reviewer APPROVED verdict. Exported to the bible.
USER-DIRECTIVE  above CONFIRMED: never decays, never agent-superseded. (Gate pending: yashr-9ha.)

INTEGRITY MACHINERY (always on)

Supersede-on-request: corrections link to the prior entry by default -- both stay live; pass supersedes to retire it for real.
Cross-type contradiction flag: same-symbol opposite claims get flagged for /pm kb-review.
Auto-staleness: prime re-hashes source files; changed code silently retires its entries.
OR-join retrieval: multi-hint primes return anything relevant, not nothing.
Stamped provenance: author/source are tool-set enums -- you can always ask who knew what.
